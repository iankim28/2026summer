# Multilingual Consensus Purification — implementation & empirical test

This repo implements the proposal in `proposal.md` and runs it end-to-end to test
whether the idea works. **Verdict: the core premise does not hold.** See
**`FINDINGS.md`** for the full write-up and **`results/summary.png`** for the figure.

### 📓 Colab notebook
**`multilingual_consensus_colab.ipynb`** is a self-contained, runnable version of the
whole experiment, tuned for a **free Colab T4** (~10–15 min end-to-end, peak GPU
memory < 6 GB). It reproduces H1 transfer, the mechanism, the disagreement detector,
H2, and the consensus denoiser + adaptive attack + ablation. Generated by
`build_notebook.py`; verified to run end-to-end and reviewed by a multi-agent pass
(`review_workflow.js`). Open in Colab → Runtime → T4 GPU → Run all.

## TL;DR
A single-language (English-only) L∞ attack on multilingual CLIP transfers
**~completely** to all other languages and makes them **agree on the wrong class**
(agreement *rises* under attack). So cross-lingual disagreement — the signal the
whole proposal depends on — does not exist. The ensemble and disagreement detector
fail; the denoiser is just generic purification that dies under adaptive attack.

## Environment
Isolated venv at `.venv` (reuses system torch 2.4 + CUDA 12.1, adds open_clip,
a compatible huggingface_hub, sentencepiece, multilingual-clip).
```bash
source .venv/bin/activate
export CUDA_VISIBLE_DEVICES=0   # any free GPU
```

## Files
| file | purpose |
|---|---|
| `mclip_lib.py` | model loading, per-language label embeddings, classification, translations |
| `data_utils.py` | STL-10 / CIFAR-10 loaders (pixel-space 224² [0,1] tensors) |
| `attacks.py` | L∞ FGSM / PGD in pixel space (single- and multi-language) |
| `run_clean.py` | clean zero-shot accuracy + clean agreement |
| `run_transfer.py` | **core**: single-language attack, transfer to all languages, saves probs |
| `analyze.py` | disagreement-detector ROC-AUC, prediction collapse |
| `embed_sim.py` | cross-lingual label-embedding geometry (the mechanism) |
| `denoiser.py` | consensus-purification denoiser: train + non-adaptive + adaptive eval; `--train_langs` for the ablation |
| `mclip_fredde.py` | confirmation on the literal FreddeFrallan M-CLIP |
| `plot.py` | summary figure |
| `results/` | JSON results, saved probs (`.npz`), denoiser weights, `summary.png` |

## Reproduce the key results
```bash
python run_clean.py    --dataset stl10 --n 1000
python run_transfer.py --dataset stl10 --n 1000 --attack pgd --steps 20 --attacked en
python run_transfer.py --dataset cifar10 --n 1000 --attack pgd --steps 40 \
       --attacked en --eps_grid 0.25,0.5,1,2,4 --tag loweps   # low-eps sweep
python analyze.py  --probs results/probs_stl10_pgd.npz
python embed_sim.py --dataset stl10
python denoiser.py --dataset stl10 --train_langs en,ko,es,fr,ja --tag consensus
python denoiser.py --dataset stl10 --train_langs en             --tag enonly   # ablation
python mclip_fredde.py --dataset stl10                          # literal M-CLIP
python plot.py
```

## Model
`open_clip xlm-roberta-base-ViT-B-32` (laion5b): frozen multilingual CLIP, shared
ViT-B/32 image encoder, XLM-R text tower. Attacks are pixel-space L∞ at the 224²
encoder input; CLIP normalization is folded into the forward pass.
